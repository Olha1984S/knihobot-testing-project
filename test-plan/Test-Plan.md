# Test Plan – Knihobot Website

## Project Overview

This project focuses on testing the publicly available Knihobot website.
The purpose of testing is to validate core website functionality, usability, navigation, and user interactions from a QA perspective.
Testing activities include both manual and automated testing approaches.


## Testing Objectives

The main objectives of testing are:
- Verify that core website functionality works correctly
- Validate navigation and page transitions
- Check search and filtering behavior
- Validate forms and user input handling
- Identify UI/UX inconsistencies
- Detect functional defects


## Test Environment

Website: https://knihobot.cz
Browser: Google Chrome 147.0.7727.139
Device: Desktop
OS: Windows 11

## Testing Types

The following testing types are included:
- Functional testing
- UI testing
- Exploratory testing
- Usability testing
- Basic automation testing


## Scope of Testing

### Included in Scope

- Homepage
- Navigation menu
- Search functionality
- Filters and categories
- Product pages
- Forms
- Internal links

### Out of Scope

- Security testing
- Load/performance testing
- Payment processing validation
- Database testing


## Automation Scope

Automation testing may include:
- Navigation scenarios
- Search validation
- Link validation
- Basic UI checks
- Form interaction

Playwright is used for automated end-to-end testing.


## Defect Management

Defects identified during testing are documented in the bug-reports section.

Each defect includes:
- title
- severity
- steps to reproduce
- expected result
- actual result
